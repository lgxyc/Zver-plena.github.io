网络核心

分组交换

  • packet switch
  • router
  • link-layer-switch

存储转发传输：

路由器在转发前需要接受，存储和处理整个分组

如果不存储，一旦比特到达就转发比特，

那么为了你当前的数据可以被完整的转发到目的地，那条线路会被一直占用；

如果你存储，那么在你存储的过程中，这条线路可以用做其他组到达的且已经存储完的数据转发

分组丢失

router 的处理队列是有容量限制的

如果massage到达的时候队列为满的，那么该router就无法将该massage入队

故而会将其丢弃

分组交换网络中的时延

  • nodal processing delay : 节点处理时延

  • queuing delay : 排队时延

  • transmission delay : 传输时延

  • propagation delay : 传播时延

  • node delay : 端到端时延就是上述有所时延总和

nodal processing delay：

检查收到的分组的首部，

查询表该讲分组发向哪里

queuing delay

到达 router 其实就是一个处理队列

取决于网络堵塞程序

对于queuing delay通常用统计测量

a: 分组到达队列的平均速率（pkt/s）

R: 链路传输路率

L: 表示分组由L bit 组成

比特到达队列的平均速率就是 La bps

La/R ： 被称为流量强度

transmission delay

transmission delay 是路由器推出分组所需要的时间

由分组长度和链路传输速率决定

与两路由器之间的距离无关

propagation delay

propagation delay ：一个比特从一台路由器传播到另一台路由器所需要的时间

propagation delay 和 transmission delay 有一些区别

电路交换

与分组交换相比，电路交换的优点：

  • 不会出现堵塞，因为资源是预先就已经分配了的
  • 适合实时服务，例如现在的电话通信

但是相较于分组交换还是比较浪费资源，现在也是大部分都转成了分组交换

电路交换有以下两种方式来进行共享

  • TDM（Time division Multiplexing)
  • FDM（Frequency-division Multiplexing）

TDM 和 FDM 的优点：

TDM是通过区分不同的频段来实现共享

对于每条电路只可以得到部分的带宽，是真真的共享

FDM则是根据时间片来实现共享

对于每条电路在他的时间片内可以得到链路的全部带宽

一：

全球ISP + 客户ISP

全球ISP（n）+ 区域ISP（n） + 客户ISP

全球ISP（n）+ 区域ISP（n）+ PoP + IXP + 多宿 + content provider network + 客户ISP

Web和HTTP

响应时间模型

往返时间RTT（round-trip time）：

一个小的分组从客户端到服务器，在回到客户端的时间（传输时间忽略）

响应时间：

  • 一个RTT 发起TCP连接
  • 一个RTT用来请求HTTP请求并等待响应
  • 文件传输时间

非持续连接

非持续链接：每个请求/响应都是经过一个单独的TCP连接来进行的

  1. 那么首先每次握手的时间都会有，每次对象都需要2个RTT，对于双方对象具有很大的时延

  2. 对于每次建立TCP连接，服务器和客户都要分配TCP的缓冲区和TCP变量，如果发送一个资源需要10个分组，那么就需要建立10个TCP连接

  3. HTTP/1.0使用非持久连接

持续连接

持续连接：每个请求/响应都是通过一个TCP连接来进行的

  1. 请求在单个持续的TCP连接上进行，采用流水线的方式发送

  2. 一般的，如果一个TCP连接超过一定的时间间隔就会关闭

  3. HTTP/1.1默认使用持久连接

Web缓存

Web cache 也叫做 proxy server

为最近请求过的对象存储起来，以待可以快速响应请求

  • 工作流程

这里服务器命名为A

  1. 客户对 A 发送TCP连接，实质上是在对proxy server 发起TCP连接
  2. proxy server 查看是否有A的副本
    1. 如果有，那么可以直接返回给客户该 A 的响应
    2. 如果没有，proxy server 会发起TCP给A ，响应结果给客户，并且留下副本
  • 好处

  1. 可以减少对客户的响应时间

  2. 减少内部网络与 Internent 接入链路上的流量

通常客户和proxy server 之间通过高速连接

目前的CDN(Content Distribution Network)，就是在Intenet上建立了很多地理上分散的缓存器。

conditional Get方法

前面说到，因为有proxy server 当你发起请求实质上是在拿去缓存，那么缓存是否是旧的呢？。

HTTP提供了一个条件Get(conditional GET)方法让proxy server 来验证他的对象是最新的,

当你请求数据的时候，proxy server会向对应的服务器发送一个GET方法

会在请求报文中包含一个“if-Modified-Since”

其中会去询问在缓存保留的数据的Date之后是否修改过

如果修改过，那么会返回修改后的数据

如果没有修改过，会返回304状态码，并且body为空，因为没有必要在发送一次

HTTP在默认方式使用的是持续连接

print("bind ： ip = %s, port = %s"%("any",serverPort))
serverSocket.bind(('',serverPort))

# tcp等待连接最大数量
serverSocket.listen(5)
# 多线程接受
def tcplink(clientSocket,clientaddress):
    while True:
        message = clientSocket.recv(2048)
        modifiedMessage = message.decode().upper()
        print("向{0}发送信息：{1}".format(clientaddress,modifiedMessage))
        # 可以直接对应的clientSocket发送
        clientSocket.send(modifiedMessage.encode())        
        if not message or message.decode() == 'exit':
            print("断开连接······")
            break
    clientSocket.close()      

print("The server is ready to receive")
while True:
    # 创建对应client的tcp
    # 握手阶段在这部分
    clientSocket,clientaddress = serverSocket.accept()
    print("接受TCP连接，client = {0}".format(clientaddress))
    t = threading.Thread(target=tcplink,args=(clientSocket,clientaddress))
    t.start()

        recvData,address = clientSocket.recvfrom(1024)

        rtt = recvTime - sendTime

        print("rtt = {0}".format(rtt))
    except timeout:
        print("request timed out")
        LOSS+=1
        
print("丢包率 = {0}".format(float(LOSS)/float(SIZE)))

数据链路层

数据链路层概述

链路（Link）就是从一个结点到相邻结点的一段物理线路，而中间没有任何其他的交换结点。

数据链路（Data Link）是指把实现通信协议的硬件和软件加到链路上，就构成了数据链路。

数据链路层以帧为单位传输 和处理数据。

数据链路层的重要问题

使用点对点信道的数据链路层

  1. 封装成帧:将上层数据封装
  2. 差错检测:通过检测码和检测算法判断误码.
  3. 可靠传输: 尽管误码是不能完全避免的，但若能实现发送方发送什么，
    接收方就能收到什么，就称为可靠传输。

使用广播信道的数据链路层

  1. 共享式以太网的媒体接入控制协议CSMA/CD
  2. 802.11局域网的媒体接入控制协议CSMA/CA

数据链路层的互连设备

  1. 网桥和交换机的工作原理
  2. 集线器（物理层互连设备）与交换机的区别

封装成帧

  • 封装成帧是指数据链路层给上层交付的协议数据单元添加帧头和帧尾使之成为帧。

    • 帧头和帧尾都有重要的控制信息

    • 帧头和帧尾的作用之一就是帧定界。

并不是每一个协议都有帧定界标志,例如PPP帧有,而以太网V2的MAC帧没有

透明传输

  • 透明传输是指数据链路层对上层交付的传输数据没有任何限制，就好像数据链路层不存在一样

    • 面向字节的物理链路使用字节填充（或称字符填充）的方法实现透明传输。

      • 内容等于帧定界符的前面加个转义字符来表示这是内容

    • 面向比特的物理链路使用比特填充的方法实现透明传输。

      • 高级数据链路控制协议HDLC采用帧头和帧尾中的标志字段作为帧定界符,其值为01111110;

HDLC为了实现“透明传输” 采用“零比特填充法”（每5个连续1后面插入一个比特0）

传输效率

  1. 为了提高帧的传输效率，应当使帧的数据部分的长度尽可能大些。

帧的数据部分长度应该 > 帧头 + 帧尾

  1. 考虑到差错控制等多种因素，每一种数据链路层协议都规定了帧的数据部分的长度上限，

即最大传送单元MTU(Maximum Transfer Unit)

差错检测

  • 计算机网络中如何选择采用差错检查

在计算机网络中通常采用我们后续中将要讨论的检错重传方式来纠正传输中的差错，或者仅仅是丢弃检测到差错的帧，这取决于数据链路层向其上层提供的是可靠传输服务还是不可靠传输服务。

  • 比特差错

实际的通信链路都不是理想的，比特在传输过程中可能会产生差错：1可能会变成0，而0也可能变成1。这称为比特差错。

  • 误码率BER

在一段时间内，传输错误的比特占所传输比特总数的比率称为误码率BER（Bit Error Rate）。

  • 差错检错码

使用差错检测码来检测数据在传输过程中是否产生了比特差错，感数据链路层所要解决的重要问题之一.

检错码只能检测出帧在传输过程中出现了差错，但并不能定位错误，因此无法纠正错误。

要想纠正传输中的差错，可以使用冗余信息更多的纠错码进行前向纠错。但纠错码的开销比较大，在计算机网络中较少使用。

奇偶校验

在待发送的数据后面添加1位奇偶校验位，使整个数据 （包括所添加的校验位在内）中“1”的个数为奇数（奇校验）或偶数（偶校验）

循环冗余校验CRC（Cyclic Redundancy Check）

  • CRC的作用

循环冗余校验CRC有很好的检错能力（漏检率非常低），虽然计算比较复杂，但非常易于用硬件实现，因此被广泛应用于数据链路层。

CRC流程

  1. 收发双方约定好一个生成多项式G（x）；

  2. 发送方基于待发送的数据和生成多项式计算出差错检测码（冗余码）
    将其添加到待传输数据的后面一起传输；

  3. 接收方通过生成多项式来计算收到的数据是否产生了误码

CRC生成多项式举例

常用的生成多项式

发送方计算余数

  • 构造被除数

待发送信息后面添加生成多项式最高次数个0

  • 构造除数

生成多项式各项系数构成的比特串

  • 做异或运算

  • 检查余数

余数的位数应与生成多项式最高次数相同，如果位数不够，则在余数前补0来凑足位数。

  • 把余数加在数据后面发送

接收方

  • 构造被除数

接收到的信息就是被除数

  • 构造除数

生成多项式各项系数构成的比特串

  • 异或运算

  • 检查余数

余数为0，可认为传输过程无误码；

余数不为0，可认为传输过程产生误码。

可靠传输

可靠传输的基本概念

  • 比特差错只是传输差错中的一种。

  • 从整个计算机网络体系结构来看，传输差错还包括分组丢失、分组失序以及分组重复。

  • 分组丢失、分组失序以及分组重复这些传输差错，一般不会出现在数据链路层，而会出现在其上层。

  • 可靠传输服务并不仅局限于数据链路层，其他各层均可选择实现可靠传输。

  • 可靠传输的实现比较复杂，开销也比较大，是否使用可靠传输取决于应用需求。

数据链路层向上层提供的服务类型

不可靠传输服务：仅仅丢弃有误码的帧，其他什么也不做；

可靠传输服务：想办法实现发送端发送什么，接收端就收到什么。

协议

  • 运输层

    • TCP向其上层提供面向连接的可靠传输服务

    • UDP向其上层提供无连接、不可靠传输服务

  • 网际层

    • IP向其上层提供无连接、不可靠传输服务

  • 网络接口层

    • 802.11无线局域网要求数据链路层实现可靠传输

    • 以太网不要求数据链路层实现可靠传输

可靠传输的实现机制

  1. 停止-等待协议SW
  2. 回退N帧协议GBN
  3. 选择重传协议SR

这三种可靠传输实现机制的基本原理并不仅限于数据链路层，可以应用到计算机网络体系结构的各层协议中。

可靠传输的实现机制–停止-等待协议SW
