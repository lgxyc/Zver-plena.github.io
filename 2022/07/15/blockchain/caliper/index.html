记录时间2022.07.15

最近因为要测试quorum网络的TPS，

自己也不知道怎么去测试js的async和监控区块链网络的transaction成功和失败，

由于上述原因，所以找到了Caliper，

但是文档很少啊，然后看着官网文档，自己英语也不太好，

弄了一整天都是爆红，就一个位置不对他都给你弄一堆错误信息，

好在最后还是启动成功了，下面记录一下自己启动成功的配置文件和一些坑点

Hyperledger Caliper

对区块链的压力测试

测试quoum

安装Caliper

npm install --only=prod     @hyperledger/caliper-cli

需要配置 network 和 beanchmark 配置文件

坑点：

因为caliper可以帮你部署合约，你需要指明contractDeveloperAddress，还有在合约里面指明合约的json文件路径path，但是我就是在这不断的报错有问题，然后排错不了，所以我手动的部署了合约在测试网络上，然后把contract的address 和 abi 写在配置文件中

并且在启动的时候加上选项--caliper-flow-only-test=true:不需要caliper实例化合约，只需要执行测试

下列就是一些参数解释：

  • url : websocket url
  • fromAddressSeed : node privateKey

还有就是要申明 test 的gas limit。

一种格式是 ‘function name’ : 100000

还有就是按照函数的类型来设置 ‘tranfer’ : 100000

具体可以看官方文档

这个配置文件的一些坑点是

你必须要有rateControl 和tps 属性

然后是一些信息的说明

  • txNumber: 发送次数

  • type: fixed-load : 发送速率

  • tps: 每秒发送个数

  • workload: 调用合约的js文件路径

具体的rateControl的支持可以看官网文档

branch/config.yaml

# rateControl https://hyperledger.github.io/caliper/v0.5.0/rate-controllers/#fixed-load

test:
  name: Fabonacci
  workers:
    number: 3
  rounds:
    - label: Fabonacci_round_load
      txNumber: 100
      rateControl:
        type: fixed-load
        opts:
          tps: 10
      workload:
        module: testScript/TestFabonacci.js
    - label: Fabonacci_round_rate
      txNumber: 1000
      rateControl:
        type: fixed-rate
        opts:
          tps: 20
      workload:
        module: testScript/TestFabonacci.js

这个就是参照官网文档写的

testScrpt/TestFabonacci.js


`use script`;

const { WorkloadModuleBase } = require('@hyperledger/caliper-core');

class FabonacciWorkload extends WorkloadModuleBase{
	// 参照直接copy ，
    constructor(){
        super();
    }     
    // 初始化的参数可以看官网文档
    async initializeWorkloadModule(workerIndex, totalWorkers, roundIndex, roundArguments, sutAdapter, sutContext)
         this.workerIndex = workerIndex;
         this.totalWorkers = totalWorkers;
         this.roundIndex = roundIndex;
         this.roundArguments = roundArguments;
         this.sutAdapter = sutAdapter;
         this.sutContext=sutContext;

     }
	// 发起交易
    async submitTransaction(){
        // 合约参数
        const Args = [];
        // contract: ’contract name‘,
        // verb : 'test function name',
        // args :  [function parameter,···],
        const arg ={
            contract: 'Fabonacci',
            verb: 'fabonacci',
            args: Args
        };
       
        // 通过sutAdapter发送请求
        await this.sutAdapter.sendRequests(arg);
    }
	// 执行完成，释放资源占用
    async cleanupWorkloadModule(){};
}

function createWorkloadModule(){
    return new FabonacciWorkload();
}
module.exports.createWorkloadModule = createWorkloadModule;

npx caliper launch  manager  \

caliper问题

目前这个还存在一些没有解决的错误

  • 去掉–caliper-flow-only-test=true 就会报错

  • 合约要自己手动部署，在networkconfig.json里面让caliper帮你自动部署会报错

不错不错，解决了一个谜

–caliper-flow-only-test=true

只需要进行测试，合约就不帮你部署了

caliper 帮我部署合约会报错，然后我就自己手动发布了合约，然后在networkconfig.json 中手动设置address 和 abi

所以我在运行的时候可以加上–caliper-flow-only-test=true
