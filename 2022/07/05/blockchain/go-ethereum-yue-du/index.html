trie（MPT）

decode

参考博客

MPT 引入了（hex-prefix,HP)编码来对key进行编码

  • RAW：原始编码

  • HEX：十六进制编码

  • HEX-Prefix：十六进制前缀编码

RAW编码

就是原生的key值，不做任何改变，这种编码方式的key，也是MPT对外提供借口的默认编码方式

例如[ cat: dog ]，Raw编码为[‘c’, ‘a’, ‘t’ ]，换成ASCLL表示方式就是[63,61,74] （十六进制）

Hex编码

为了减少分支节点的孩子个数，需要对key的编码进行转换

将key的高低四位拆成两个字节进行存储

Raw -> Hex 编码的规则就是

  • 将Raw编码的每个字符，根据高4位低4位拆成两个字节；
  • 该节点是叶子节点，则在末位添加一个ASCII值为16的字符作为终止标志符；
  • 如果是扩展节点，不加任何字符

例如[ cat: dog ]，Hex编码为[3, 15, 3, 13, 4, 10]

输入”cat”，Raw编码后 [63, 61, 74]

63 / 16 = 3
63 % 16 = 15

61 / 16 =3

61 % 16 =13

74 / 16 = 4
74 % 16 = 10
编码后的结果 [3，15，3，13，4，10]

如果是叶子节点[3，15，3，13，4，10,16]

Hex-Prefix编码(Hp编码 )

之前说的Hex编码后的数据是在内存里面的，如果要对Hex编码数据进行持久化

那么就会发现一个问题我们对原数据进行了扩展，本来1个byte的数据被我们变成了2个byte，显然这对于存储来说是存储空间变大了一倍的，于是就又有了HP编码。

HP编码的过程如下；

输入key（Hex编码的结果）如果有标识符，则去掉这个标识符。

key的头部填充一个nibble，填充的规则如下

  • 如果key的nibble长度是偶数则最后一位 0

  • 如果key的nibble长度是奇数则最后一位 1

  • 如果key是扩展节点则倒数第二位是 0

  • 如果key是叶子节点则倒数第二位是 1

例如[ cat: dog ]，Hex编码为[3, 15, 3, 13, 4, 10]

如果是叶子节点[3，15，3，13，4，10,16]，这里我们按照叶子节点来处理

HP编码

1.nibble 长度为7，所以长度是奇数

2.去掉16，同时表明这个是叶子节点

综上两个条件得到hp编码为 0010 0000

将HP编码后的结果二进制表示

[0010, 0000，0011, 1111, 0011, 1101, 0100, 1010]

将编码后的结果合并后

[00100000，00111111, 00111101, 01001010]

转换为十进制后

[32,63,61,74]

三种编码转换

Compact就是上面说的HEX-Prefix

keybytes为按完整字节（8bit）存储的正常信息

hex为按照半字节nibble（4bit）储存信息的格式。

go-ehereum/trie/encoding

func compactToHex(compact []byte) []byte {
	if len(compact) == 0 {
		return compact
	}
	base := keybytesToHex(compact)
	// delete terminator flag
	if base[0] < 2 {
		base = base[:len(base)-1]
	}
	// apply odd flag
	chop := 2 - base[0]&1
	return base[chop:]
}

func hexToCompact(hex []byte) []byte {
	terminator := byte(0)
	if hasTerm(hex) {
		terminator = 1
		hex = hex[:len(hex)-1]
	}
	buf := make([]byte, len(hex)/2+1)
	buf[0] = terminator << 5 // the flag byte
	if len(hex)&1 == 1 {
		buf[0] |= 1 << 4 // odd flag
		buf[0] |= hex[0] // first nibble is contained in the first byte
		hex = hex[1:]
	}
	decodeNibbles(hex, buf[1:])
	return buf
}

func keybytesToHex(str []byte) []byte {
	l := len(str)*2 + 1
	var nibbles = make([]byte, l)
	for i, b := range str {
		nibbles[i*2] = b / 16
		nibbles[i*2+1] = b % 16
	}
	nibbles[l-1] = 16
	return nibbles
}

func hexToKeybytes(hex []byte) []byte {
	if hasTerm(hex) {
		hex = hex[:len(hex)-1]
	}
	if len(hex)&1 != 0 {
		panic("can't convert hex key of odd length")
	}
	key := make([]byte, len(hex)/2)
	decodeNibbles(hex, key)
	return key
}

bloom Filter

设计

bloom 为了消除不必要的磁盘访问。

误报分析

虽然存在误报的风险，但相较于其他的数据结构，bloom filter有着空间和时间的优势

实际上，哈希表就是k = 1 的bloom filter

最佳的k值的bloom filter

可以并行查找

误报的概率计算

参考WIki里面的计算公式

m = BIT_SIZE

n = ADD_SIZE

k = HASH_SIZE

插入一个元素后，某位为 1 的概率为

$\cfrac{1}{m}$

插入一个元素后，某位为 0 的概率为

$1-\cfrac{1}{m}$

如果有有k个hash函数,k个hash函数 no significant correalation

插入一个元素后，某位为 0 的概率为

$(1-\cfrac{1}{m})^k$

for large m

$\lim_{m->+\infty}(1-\cfrac{1}{m})^m = \cfrac{1}{e}$

for large m

$(1-\cfrac{1}{m})^k =((1-\cfrac{1}{m})^m)^{\cfrac{k}{m}}\approx e^{\cfrac{-k}{m}}$

插入n个元素后，某位为 0 的概率为

$(1-\cfrac{1}{m})^{kn} \approx e^{\cfrac{-kn}{m}} $

插入n个元素后，某位为 1 的概率为

$1-(1-\cfrac{1}{m})^{kn} \approx 1 -e^{\cfrac{-kn}{m}} $

已经插入了n个元素之后，查询一个没有添加的元素在集合中的概率

则某位为1的概率为上述的$1-(1-\cfrac{1}{m})^{kn} \approx 1 -e^{\cfrac{-kn}{m}}$

那么k位都为1的概率为 $p = (1-(1-\cfrac{1}{m})^{kn})^k \approx (1 -e^{\cfrac{-kn}{m}})^k$

误报率：

$p = (1-(1-\cfrac{1}{m})^{kn})^k \approx (1 -e^{\cfrac{-kn}{m}})^k$

最佳数量的hash 函数

$k = \cfrac{m}{n}\ln2$

对于给出的 n 和 误报率(p) 求 m，在假设最佳数量的hash函数条件下

将 $k = \cfrac{m}{n}\ln2$ 代入到误报率公式$p \approx (1 -e^{\cfrac{-kn}{m}})^k$，得

$\ln p = \cfrac{m}{n}(\ln 2)^2$

求得m为

$m = - \cfrac{n \ln p}{(\ln 2)^2}$

带入 $m = - \cfrac{n \ln p}{(\ln 2)^2}$ 到 $k = \cfrac{m}{n}\ln2$，得

$k = \cfrac{-\ln p}{\ln 2} = -\log_2 p$

这意味着对于给定的误报率p，bloom filter的空间m和添加元素n成正比例，

所需的hash函数数量取决于误报率p

一些公式计算

package main

import (
	"fmt"
	"math"
)

func main() {
	m := 2048.0 // 位数
	n := 300.0  // 插入的元素个数
	e := 2.72   // e的近似值
	k := 3.0    // hash函数个数

	// 对于给出的 m ， n ， k 的误报概率
	fmt.Printf("误差 =  %v\n", math.Pow(1.-math.Pow(e, -k*n/m), k))
	// 对于给出的 m 和 n 最佳的hash个数
	fmt.Printf("最佳的hash个数 = %v\n", 1.0*m/n*math.Ln2)
	// 对于给出的 n 和 误报概率 给出最佳的 m，这里的最佳的 m 在假设使用了对应的最佳的 hash 个数
	fmt.Printf("假设误差为0.01，n = 500，并且假设使用最佳的 k，那么对于需要的最佳位数 m = %v\n", -math.Log2(0.01)/math.Ln2*n)
	fmt.Printf("对于上述的误差为0.01,对应的最佳的 k = %v\n", -math.Log2(0.01))
	m = -1.44 * math.Log2(0.01) * n
	//计算出来的结果应该和上述的结果相差不大
	fmt.Printf("对于上述计算出来的最佳 m 和给出的 n 又可以计算出最佳的 k = %v\n", m/n*math.Ln2)

}

package bloom

import (
	"encoding/binary"
	"github.com/ethereum/go-ethereum/crypto"
	"golang.org/x/crypto/sha3"
	"sync"
)

const (
	BloomByteLength = 256
	BloomBitLength  = 8 * BloomByteLength
)

type Bloom [BloomByteLength]byte

// find
func (b *Bloom) find(topic []byte) bool {
	i1, v1, i2, v2, i3, v3 := bloomValues(topic, make([]byte, 6))
	return v1 == v1&b[i1] &&
		v2 == v2&b[i2] &&
		v3 == v3&b[i3]
}
// Add 内部实现的是add
func (b *Bloom) Add(d []byte) {
	b.add(d, make([]byte, 6))
}
// add
func (b *Bloom) add(d []byte, buf []byte) {
	i1, v1, i2, v2, i3, v3 := bloomValues(d, buf)
	b[i1] |= v1
	b[i2] |= v2
	b[i3] |= v3
}

var hasherPool = sync.Pool{
	New: func() interface{} { return sha3.NewLegacyKeccak256() },
}

// return index
func bloomValues(data []byte, hashbuf []byte) (uint, byte, uint, byte, uint, byte) {

	sha := hasherPool.Get().(crypto.KeccakState)

	//生成散列函数，初始化
	sha.Reset()
	//对data进行加密
	sha.Write(data)
	//加密后的结果输入的 hashbuf
	sha.Read(hashbuf)
	//保留散列值
	hasherPool.Put(sha)

	// The actual bits to flip
	// 翻转的bit位
	// 最多左移7位
	v1 := byte(1 << (hashbuf[1] & 0x7))
	v2 := byte(1 << (hashbuf[3] & 0x7))
	v3 := byte(1 << (hashbuf[5] & 0x7))

	// The indices for the bytes to OR in
	// index 在bloom数组中
	i1 := BloomByteLength - uint((binary.BigEndian.Uint16(hashbuf)&0x7ff)>>3) - 1
	i2 := BloomByteLength - uint((binary.BigEndian.Uint16(hashbuf[2:])&0x7ff)>>3) - 1
	i3 := BloomByteLength - uint((binary.BigEndian.Uint16(hashbuf[4:])&0x7ff)>>3) - 1

	//fmt.Printf("下面是对应的[k,v]-------------------------------------\n")
	//fmt.Printf("%v = %v, %v = %v, %v = %v \n", i1, v1, i2, v2, i3, v3)

	return i1, v1, i2, v2, i3, v3
}

func TestBloom2(t *testing.T) {
	addSize := int(500)
	findSize := int(1e5)
	len := 20

	var bloom Bloom
	for i := 1; i <= addSize; i++ {
		rand.Seed(time.Now().UnixNano())
		bloom.Add(randC1(len))
	}
	count := 0
	for i := 1; i <= findSize; i++ {
		rand.Seed(time.Now().UnixNano())

}
